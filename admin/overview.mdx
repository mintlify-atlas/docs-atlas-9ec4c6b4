## Understanding Admin Access

VoteLab uses owner-based access control to protect administrative functions. Only the contract owner can manage elections, control workflow states, and withdraw accumulated funds.

### Owner Verification

The system verifies admin access by comparing your connected wallet address with the contract owner address:

```typescript
// From useMemberVote.ts:104
isOwner: address?.toLowerCase() === (ownerAddress as string)?.toLowerCase()
```

This check happens automatically when you connect your wallet. The `getOwner()` contract function returns the authorized admin address:

```typescript
// Contract ABI function signature
{
  "type": "function",
  "name": "getOwner",
  "inputs": [],
  "outputs": [
    {
      "name": "",
      "type": "address",
      "internalType": "address"
    }
  ],
  "stateMutability": "view"
}
```

<Warning>
  Admin functions trigger the `MemberVote__NotOwner` error if called by non-owner addresses. Always verify you're connected with the correct wallet.
</Warning>

## Admin Panel Interface

When you connect as the contract owner, VoteLab displays an admin console at the bottom of the screen:

```typescript
// From AdminPanel.tsx:14
if (!isOwner) return null;
```

The panel only renders for verified owners, providing a clean interface for non-admin users.

### Panel Features

<Steps>
  <Step title="Workflow Status Display">
    Shows the current election phase: REGISTERING (0), VOTING (1), or RESETTING (2)
  </Step>
  <Step title="Phase Control Buttons">
    Context-aware controls that appear based on the current workflow state
  </Step>
  <Step title="Prize Pool Monitor">
    Real-time display of accumulated ETH from voter entry fees
  </Step>
  <Step title="Fund Withdrawal">
    One-click button to collect funds when the prize pool has a balance
  </Step>
</Steps>

## Admin State Management

The admin panel hooks into contract state through the `useMemberVote` custom hook:

```typescript
// From AdminPanel.tsx:9-12
const { 
  workflowStation, prizePool, isOwner, 
  startVote, resetVotes, withdrawFunds, isPending 
} = useMemberVote();
```

### Key Admin Properties

<ParamField path="workflowStation" type="number">
  Current election phase: 0 (REGISTERING), 1 (VOTING), or 2 (RESETTING)
</ParamField>

<ParamField path="prizePool" type="string">
  Total ETH in contract, formatted with `formatEther()` for display
</ParamField>

<ParamField path="isOwner" type="boolean">
  Boolean flag indicating if connected address matches contract owner
</ParamField>

<ParamField path="isPending" type="boolean">
  Transaction pending status, disables buttons during blockchain operations
</ParamField>

## Workflow Constants

VoteLab uses numeric constants to represent election phases:

```typescript
// From AdminPanel.tsx:6
const WORKFLOW = { REGISTERING: 0, VOTING: 1, RESETTING: 2 };
```

These map directly to the smart contract's `WorkFlowStation` enum, ensuring UI state matches on-chain state.
