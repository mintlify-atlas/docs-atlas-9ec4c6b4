## Prerequisites

Before you begin, ensure you have the following installed on your system:

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the VoteLab repository from GitHub:

    ```bash
    git clone https://github.com/Pragyat-Nikunj/Voting-Dapp.git
    cd Voting-Dapp
    ```
  </Step>

  <Step title="Navigate to Frontend Directory">
    Move into the frontend application directory:

    ```bash
    cd frontend
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    This will install key dependencies including:
    - **Next.js 16.1.6** - React framework with App Router
    - **Wagmi v2.19.5** - Web3 React hooks
    - **Viem v2.46.2** - TypeScript Ethereum library
    - **RainbowKit v2.2.10** - Wallet connection UI
    - **Tailwind CSS v4** - Utility-first styling
    - **Framer Motion v12** - Animation library
  </Step>

  <Step title="Start Development Server">
    Launch the local development server:

    ```bash
    npm run dev
    ```

    The application will be available at `http://localhost:3000`
  </Step>
</Steps>

## Project Structure

VoteLab follows Next.js 16 App Router conventions:

```
frontend/
├── src/
│   ├── app/              # Next.js App Router pages
│   ├── components/       # React components
│   │   ├── AdminPanel.tsx
│   │   └── DashboardGrid.tsx
│   └── hooks/            # Custom React hooks
│       └── useMemberVote.ts
├── test/
│   ├── components/       # Component tests
│   └── setup.ts          # Vitest configuration
├── package.json
├── next.config.ts
├── vitest.config.ts
└── tsconfig.json
```

## Configuration Files

### Next.js Configuration

The project uses React Compiler for optimized performance:

```typescript title="next.config.ts"
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  reactCompiler: true,
};

export default nextConfig;
```

### TypeScript

Path aliases are configured for clean imports:

```typescript
import { AdminPanel } from '@/components/AdminPanel';
import { useMemberVote } from '@/hooks/useMemberVote';
```

<Note>
  The `@` alias maps to the `./src` directory for convenient imports.
</Note>

## Smart Contract Setup

<Warning>
  Smart contract development requires Foundry to be installed separately.
</Warning>

For smart contract development and testing:

<Steps>
  <Step title="Install Foundry">
    Follow the [Foundry installation guide](https://book.getfoundry.sh/getting-started/installation):

    ```bash
    curl -L https://foundry.paradigm.xyz | bash
    foundryup
    ```
  </Step>

  <Step title="Access Contract Repository">
    The smart contract is maintained in a separate repository:

    ```bash
    git clone https://github.com/Pragyat-Nikunj/Smart-Voting-Contract.git
    ```
  </Step>

  <Step title="Compile Contracts">
    ```bash
    forge build
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Node version issues">
    Ensure you're using Node.js v18 or higher:

    ```bash
    node --version
    ```

    Consider using [nvm](https://github.com/nvm-sh/nvm) to manage Node versions:

    ```bash
    nvm install 18
    nvm use 18
    ```
  </Accordion>

  <Accordion title="Dependency conflicts">
    If you encounter dependency issues, try clearing the cache:

    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="Port already in use">
    If port 3000 is occupied, specify a different port:

    ```bash
    PORT=3001 npm run dev
    ```
  </Accordion>
</AccordionGroup>
