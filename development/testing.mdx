## Testing Philosophy

VoteLab maintains high code coverage to ensure reliability in financial transactions. The testing strategy covers:

- **Frontend Components**: UI state management and user interactions
- **Smart Contracts**: On-chain logic, security, and state transitions
- **Integration**: Wallet connections and contract interactions

<Note>
  VoteLab uses a dual testing approach: **Vitest** for frontend and **Foundry** for smart contracts.
</Note>

## Frontend Testing (Vitest)

### Overview

The frontend test suite uses:
- **Vitest v4.0.18** - Fast unit test framework
- **React Testing Library v16.3.2** - Component testing utilities
- **jsdom v28.1.0** - Browser environment simulation

### Configuration

Vitest is configured with jsdom environment and path aliases:

```typescript title="vitest.config.ts"
import { defineConfig } from 'vitest/config';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
    globals: true,
    setupFiles: './test/setup.ts',
  },
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
});
```

### Test Setup

The setup file configures testing utilities and mocks:

```typescript title="test/setup.ts"
import '@testing-library/jest-dom';
import { vi } from 'vitest';

// Mock window.matchMedia for components using media queries
Object.defineProperty(window, 'matchMedia', {
  writable: true,
  value: vi.fn().mockImplementation(query => ({
    matches: false,
    media: query,
    onchange: null,
    addListener: vi.fn(),
    removeListener: vi.fn(),
    addEventListener: vi.fn(),
    removeEventListener: vi.fn(),
    dispatchEvent: vi.fn(),
  })),
});
```

## Testing Strategies

### 1. Hook Mocking

VoteLab tests mock the `useMemberVote` hook to simulate various contract states:

```typescript title="test/components/AdminPanel.test.tsx"
import { render, screen } from '@testing-library/react';
import { AdminPanel } from '@/components/AdminPanel';
import { useMemberVote } from '@/hooks/useMemberVote';
import { vi, describe, it, expect, type Mock } from 'vitest';

const mockedUseMemberVote = useMemberVote as Mock;

// Mock the entire hook module
vi.mock('@/hooks/useMemberVote', () => ({
  useMemberVote: vi.fn(),
}));

describe('AdminPanel Visibility', () => {
  it('should return null if the user is NOT the owner', () => {
    mockedUseMemberVote.mockReturnValue({
      isOwner: false,
      workflowStation: 1,
    });

    const { container } = render(<AdminPanel />);
    expect(container.firstChild).toBeNull();
  });

  it('should show Admin Console if the user IS the owner', () => {
    mockedUseMemberVote.mockReturnValue({
      isOwner: true,
      workflowStation: 1,
      prizePool: '0',
    });

    render(<AdminPanel />);
    expect(screen.getByText(/Admin Console/i)).toBeInTheDocument();
  });
});
```

<Tip>
  Mocking hooks allows testing UI behavior without connecting to a blockchain.
</Tip>

### 2. Access Control Testing

Verifies that the `AdminPanel` is physically absent from the DOM for unauthorized wallets:

```typescript
it('should return null if the user is NOT the owner', () => {
  mockedUseMemberVote.mockReturnValue({
    isOwner: false,
    workflowStation: 1,
  });

  const { container } = render(<AdminPanel />);
  expect(container.firstChild).toBeNull();
});
```

**Why this matters:**
- Prevents unauthorized access to admin controls
- Ensures security at the UI layer
- Tests owner validation logic

### 3. User Interaction Testing

Simulates user clicks to verify that `castVote` is triggered with the correct option ID:

```typescript title="test/components/DashboardInteractions.test.tsx"
import { render, screen, fireEvent } from '@testing-library/react';
import { DashboardGrid } from '@/components/DashboardGrid';
import { useMemberVote } from '@/hooks/useMemberVote';
import { vi, describe, it, expect, type Mock } from 'vitest';

const mockedUseMemberVote = useMemberVote as Mock;

vi.mock('@/hooks/useMemberVote', () => ({
  useMemberVote: vi.fn(),
}));

describe('DashboardGrid Interactions', () => {
  it('calls castVote when the Sweet Banana button is clicked', () => {
    const mockCastVote = vi.fn();

    mockedUseMemberVote.mockReturnValue({
      workflowStation: 1,
      optionAVotes: 0,
      optionBVotes: 0,
      prizePool: '0',
      isOwner: false,
      hasVoted: false,
      castVote: mockCastVote,
    });

    render(<DashboardGrid />);
    const buttons = screen.getAllByRole('button');
    const bananaButton = buttons[1];
    fireEvent.click(bananaButton);

    expect(mockCastVote).toHaveBeenCalledWith(1);
    expect(mockCastVote).toHaveBeenCalledTimes(1);
  });
});
```

**Test coverage:**
- Button click handlers work correctly
- Correct vote option ID is passed
- Functions are called exactly once

Tests ensure the UI remains a faithful representation of blockchain state:

## Best Practices

<Steps>
  <Step title="Write Tests First">
    Follow Test-Driven Development (TDD) principles:
    1. Write a failing test
    2. Implement the feature
    3. Ensure the test passes
  </Step>

  <Step title="Mock External Dependencies">
    Always mock:
    - Wallet connections
    - Contract read/write hooks
    - Network requests
  </Step>

  <Step title="Test Edge Cases">
    Cover scenarios like:
    - Empty states
    - Maximum values
    - Unauthorized access
    - Network failures
  </Step>

  <Step title="Maintain High Coverage">
    Aim for:
    - 80%+ line coverage
    - 100% coverage for critical paths (voting, admin actions)
  </Step>
</Steps>

### Common Issues

<AccordionGroup>
  <Accordion title="Hook mocking not working">
    Ensure the mock is defined before imports:

    ```typescript
    vi.mock('@/hooks/useMemberVote', () => ({
      useMemberVote: vi.fn(),
    }));

    // Then import the component
    import { AdminPanel } from '@/components/AdminPanel';
    ```
  </Accordion>

  <Accordion title="Component not rendering">
    Check if required providers are wrapped:

    ```typescript
    render(
      <WagmiProvider>
        <QueryClientProvider>
          <YourComponent />
        </QueryClientProvider>
      </WagmiProvider>
    );
    ```
  </Accordion>

  <Accordion title="Async state updates">
    Use `waitFor` for async operations:

    ```typescript
    import { waitFor } from '@testing-library/react';

    await waitFor(() => {
      expect(screen.getByText('Loaded')).toBeInTheDocument();
    });
    ```
  </Accordion>
</AccordionGroup>
